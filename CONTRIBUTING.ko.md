# 함께 번역하기

**한눈에 보는 성경 이야기**의 목표는 지구상 **모든 언어**로 성경의 큰 그림과 복음을 전하는 것입니다. 당신의 언어를 더하거나 다듬어 주세요. 🙏

**코딩 지식은 필요 없고, 번역(텍스트)만** 손보면 됩니다. 빌드·이미지·구절 링크·배포는 관리자가 합니다.

## 딱 두 가지 규칙 (이게 전부)
1. **성경 인용은 그대로(verbatim).** 당신 언어의 공인 번역본([유버전](https://www.bible.com) 등)에서 **복사**하세요. 절대 의역하거나 새로 번역하지 마세요.
2. **키·HTML 태그·구조는 그대로.** 값(따옴표 오른쪽 문장)만 번역합니다. 키 이름을 바꾸거나 `<b>`, `<p>`, `<br>` 같은 태그를 지우지 마세요. *(팩에 `books`/`yv`/`bookopt` 필드가 이미 있으면 건드리지 마세요 — 메인테이너 도구가 자동 생성·관리하는 구절-링크 데이터입니다.)*

아래는 전부 *‘어떻게’* 에 대한 설명일 뿐입니다.

## 길을 고르세요

### A · 이미 있는 언어 문구 고치기 — 가장 쉬움, 설치 불필요
1. GitHub에서 `i18n/<코드>.json` 을 엽니다. (예: 스와힐리어 = `i18n/sw.json`)
2. 오른쪽 위 **연필(✏️ Edit)** 을 누릅니다.
3. 위 두 규칙을 지키며 문구를 고칩니다.
4. **Commit changes → Propose changes → Create pull request.** 끝.

### B · 새 언어 추가하기 — 손으로
1. 당신 언어의 **코드**를 찾습니다. (아래 *코드 찾기* 참고. 예: `vi`, `sw`, `am`)
2. **`i18n/en.json` 을 복사**해 `i18n/<당신코드>.json` 으로 만듭니다. (영어가 번역 출발점입니다)
3. 모든 **값(문장)** 을 번역하되 구조는 그대로 둡니다.
   - `htmlLang` 을 당신 코드로, `ui.version` 을 당신이 쓴 성경 번역본 이름으로 설정합니다.
   - **성경 인용은 verbatim** — 실제 번역본에서 복사하세요.
   - 개수를 그대로: `epochs` 13 · `core` 7 · `love` 13 · `mis` 13 (8번과 12번은 `null` 유지).
4. **Pull Request** 를 올립니다. 책이름 구절 링크·OG 이미지·검증은 관리자가 처리합니다.

### C · Claude / AI로 — 가장 빠름
[Claude](https://claude.ai) / [Claude Code](https://www.claude.com/product/claude-code)를 쓰면, 번역 초안부터 **성경 인용 verbatim 검증**까지 Claude가 해줍니다 — 이 사이트를 만든 방식 그대로입니다. 스킬과 검증 도구는 이미 리포 안에 있고(`.claude/skills/add-language/`), Node.js + Claude Code만 있으면 됩니다.

1. 리포를 포크·클론하고 **Claude Code로 엽니다.** 처음 한 번 `npm install` 실행(qrcode 도구 설치).
2. 아래 프롬프트를 붙여넣습니다 (빈칸 두 개만 채우세요):

   > **«언어명»** 을 `/add-language` 스킬로 이 사이트에 추가해줘.
   > 성경 판본: **«유버전 판본명 / ID»**.
   > `i18n/en.json` 을 출발점으로 초안을 만들고, 모든 성경 인용은 `fetch-verse` 로 verbatim 추출하고, `validate` / `verify-verbatim` / `verify-inline` 로 깨끗해질 때까지 자가 검증한 뒤 PR을 올려줘.

3. Claude가 `i18n/<코드>.json` 초안을 만들고 인용을 verbatim으로 추출·자가검증합니다. 검토 후 PR을 올리세요.

> **중요:** Claude는 **인용 verbatim·구조는 객관적으로 정확히** 맞추지만, **저자원 언어의 자연스러움은 보증하지 못합니다.** 가능하면 그 언어 **원어민이 한 번 검수**해 주세요. (`.claude/skills/add-language/lib/native-review-prompt.md` 프롬프트 재사용 가능)

⭐ **가장 좋은 건 둘의 결합** — Claude가 초안+verbatim 검증, 원어민이 자연스러움 검수.

## 꼭 지켜주세요 (원칙)
- **관점**: 한국 개신교 대다수가 공유하는 **복음주의·개혁주의 구속사(救贖史)** 관점.
- **성경 인용 = verbatim** — 우리는 성경을 의역하지 않습니다.
- **따뜻하게**: 민감한 FAQ(공정성·값싼 용서·“착하게 살면”)는 완화된 어조를 유지하고, 특정 영화·사건을 언급하지 않습니다.
- 참조(장:절)는 가능하면 아라비아 숫자를 사용합니다.

## 코드 찾기
파일 이름이 `i18n/<코드>.json` 이므로 코드를 정확히 골라야 합니다 — 브라우저·유버전이 쓰는 표준 코드(**BCP-47 = ISO 639**)와 동일합니다.
1. **2글자(ISO 639-1)가 있으면 그걸 씁니다** (`ko`, `sw`, `vi`). 없으면 **3글자(ISO 639-3)** (`ceb`, `tpi`, `quc`).
2. 문자가 여럿인 언어만 `-` 뒤에 4글자 문자코드(`zh-Hans`, `zh-Hant`); 지역 구분이 필요할 때만 지역코드(`pt-BR`). 소문자 우선(문자코드는 첫 글자 대문자).
3. 위키백과 “List of ISO 639 codes” 나 SIL 등록부 <https://iso639-3.sil.org/code_tables/639/data> 에서 찾습니다.
4. **이미 있나요?** `i18n/` 폴더나 사이트 🌐 메뉴를 확인하세요 — 있으면 A 경로로 수정하면 됩니다.
5. **모르겠으면** 그냥 [이슈](../../issues)를 열어 주세요. 코드를 정해 드립니다.

## 파일 구조 한눈에
```jsonc
{
 "menuName": "Kiswahili",           // 메뉴에 보일 언어 이름
 "htmlLang": "sw",                  // 당신 코드
 "dir": "ltr",                      // "ltr" 또는 "rtl" (아랍어·히브리어 등은 rtl)
 "ui": { "version": "(SUV)", ... }, // version = 사용한 성경 번역본 이름
 "s": { "hero.title": "...", ... }, // 화면 UI 문구
 "epochs": [ /* 13개 장면 */ {
    "q": "성경 인용 — verbatim!",
    "cite": "Mwanzo 1:1",           // 출처 (책 장:절)
    "detail": "<p>...</p>"          // HTML 태그 보존
 } ],
 "core": [ /* 7 */ ], "love": [ /* 13 */ ], "mis": [ /* 13, 8·12번은 null */ ]
}
```

## 자동 검증
PR을 열면 GitHub Action이 **바뀐 언어만** 자동 점검합니다 — 구조·키·HTML(게이트)과 성경 인용 verbatim 대조(참고용). **다 통과 못 해도 괜찮습니다. 관리자가 함께 봅니다.**
